# Pro Casa — Бизнес-логика системы

## Что это за проект

Pro Casa (pro.casa.kz) — CRM-система для рынка недвижимости Казахстана. Основная услуга — сопровождение продажи квартир: от первого контакта с продавцом до завершения сделки и получения комиссии.

---

## Типы пользователей (роли)

В системе 5 ролей. Каждая определяет что пользователь видит в меню и что может делать через API.

### 1. ADMIN — Администратор

Полный контроль над системой.

Доступные разделы:
- Главная (Dashboard) — общая статистика по всей системе
- Сделки (CRM) — видит все сделки всех брокеров
- Стратегии (CASA) — справочник стратегий продажи
- Мои объекты — все объекты в системе
- Новостройки (Каталог ЖК, Шахматка)
- Ипотека — калькулятор
- Профиль
- Формы — управление лид-формами
- Настройки — воронки, кастомные поля
- Архив
- **Управление** (только ADMIN):
  - Пользователи — создание/редактирование брокеров, застройщиков, агентств
  - Курсы — обучающие материалы для брокеров
  - Все проекты — управление ЖК
  - Настройки AI — конфигурация DeepSeek API

Уникальные возможности:
- Создание и управление пользователями любых ролей
- Доступ к данным всех брокеров (клиенты, сделки, объекты)
- Управление ипотечными программами
- Настройки системы (AI ключи и т.д.)
- Назначение кураторов брокерам

### 2. BROKER — Брокер (основная рабочая роль)

Главный пользователь системы. Ведёт клиентов от первого контакта до закрытия сделки.

Доступные разделы:
- Главная — личная статистика (сделки, клиенты, баланс)
- Сделки (CRM) — Kanban-доска с воронками продавцов и объектов
- Стратегии (CASA) — справочник стратегий
- Мои объекты — список своих объектов недвижимости
- Новостройки (Каталог ЖК, Шахматка)
- Ипотека
- Профиль (задачи, баланс, курсы, уведомления)
- Формы — лид-формы
- Настройки — свои воронки и кастомные поля
- Архив

Что делает брокер:
- Создаёт продавцов (карточки клиентов-продавцов)
- Проводит интервью, собирает данные о продавце и объекте
- Создаёт объекты недвижимости с детальными характеристиками
- Система автоматически рассчитывает: класс объекта, ликвидность, стратегию продажи
- Ведёт продавца по воронке: Контакт → Интервью → Стратегия → Договор
- Ведёт объект по воронке: Создан → Подготовка → Лиды → Показы → Сделка → Постсервис
- Создаёт покупателей и организует показы
- Оформляет сделки, фиксирует комиссию
- Загружает документы и фото объектов
- Видит только свои данные (клиенты, сделки, объекты)

### 3. DEVELOPER — Застройщик

Управляет жилыми комплексами (новостройками) и квартирами в них.

Доступные разделы:
- Главная
- Сделки (CRM)
- Стратегии (CASA)
- Новостройки (Каталог ЖК, Шахматка)
- Ипотека
- Профиль
- Настройки
- Архив

Уникальные возможности:
- Создание и редактирование проектов (ЖК): название, город, район, класс, статус строительства, фото, видео
- Управление квартирами в своих ЖК (шахматка): номер, этаж, комнаты, площадь, цена, статус
- Видит только свои проекты

### 4. REALTOR — Риелтор

Работает аналогично брокеру, но может быть частью агентства (привязан к AGENCY через curatorId).

Доступные разделы:
- Главная
- Сделки (CRM)
- Мои объекты
- Клиенты (список продавцов)
- Новостройки
- Ипотека
- Профиль
- Настройки
- Архив

Возможности:
- Те же что у брокера: создание продавцов, объектов, покупателей, сделок
- Видит только свои данные
- Может быть подчинён агентству (AGENCY видит его статистику)

### 5. AGENCY — Агентство недвижимости

Управляет командой риелторов.

Доступные разделы:
- Главная
- Сделки (CRM)
- **Команда** (только AGENCY) — список своих риелторов с их статистикой
- Мои объекты
- Клиенты
- Новостройки
- Ипотека
- Профиль
- Формы
- Настройки
- Архив

Уникальные возможности:
- Раздел "Команда": просмотр риелторов, их сделок, продавцов, объектов
- Создание риелторов (REALTOR) в своей команде
- Видит данные своих риелторов

## Основные бизнес-сущности

### Продавец (Seller)
Клиент, который хочет продать квартиру. Содержит:
- Контактные данные (ФИО, телефон, email, ИИН)
- Причина продажи и сроки
- Ценовые ожидания (желаемая цена, минимальная цена)
- Планы после продажи (покупка, ипотека)
- Финансовые возможности
- Готовность работать с Casa
- Этап воронки продавца

### Объект недвижимости (CrmProperty)
Квартира, привязанная к продавцу. Содержит 50+ параметров:
- Базовые: комнаты, площадь, этаж, год постройки
- Конструктив: тип дома, потолки, планировка, санузел, балкон
- Локация: район, качество локации, вид из окна
- Дом: лифты, лобби, МОП, паркинг, закрытая территория
- Юридический статус: залог, обременения
- Цена: запрашиваемая, рыночная
- **Автоматически рассчитываемые поля:**
  - Класс объекта (Бизнес / Комфорт+ / Комфорт / Эконом / Старфонд)
  - Балл ликвидности (0-100)
  - Уровень ликвидности (Высокая / Выше средней / Средняя / Ниже средней / Низкая)
  - Стратегия продажи (S1-S12)
  - Тип финансирования (ипотека доступна / ограничена / только наличные)

### Покупатель (Buyer)
Потенциальный покупатель объекта:
- ФИО, телефон
- Бюджет (мин/макс)
- Предпочтения
- Статус (NEW, CONTACTED, SHOWING, NEGOTIATION, DEAL, REJECTED)

### Показ (Show)
Просмотр объекта покупателем:
- Дата, статус
- Обратная связь, оценка (1-5)

### Сделка (Deal)
Оформленная продажа:
- Сумма, комиссия брокера, комиссия Casa
- Стадия Kanban: Консультация → Договор → Продвижение → Показы
- Статус: В работе / Завершена / Отменена

### Проект (Project) — ЖК
Жилой комплекс от застройщика:
- Название, город, район, адрес, координаты
- Класс, статус строительства, дата сдачи
- Застройщик, контакты
- Ипотечные программы, акции
- Фото, видео

### Квартира (Apartment) — в ЖК
Квартира в шахматке новостройки:
- Номер, этаж, комнаты, площадь, цена
- Статус: Свободна / Забронирована / Продана
- Планировка (изображение)

### Бронирование (Booking)
Бронь квартиры в новостройке:
- Клиент, квартира, брокер
- Статус, срок действия

## Воронки (ключевой бизнес-процесс)

### Воронка продавца
Цель: довести продавца до подписания договора.

```
Контакт → Интервью → Стратегия → Договор → [Отмена]
```

1. **Контакт** — зафиксировано обращение, базовые данные
2. **Интервью** — полный сбор данных о продавце И объекте. Без заполнения — дальше не пройти
3. **Стратегия** — система рассчитывает стратегию на основе данных. Согласование с продавцом
4. **Договор** — подписан эксклюзивный договор. Активируется воронка объекта
5. **Отмена** — отказ (клиент отказался / мы отказались)

### Воронка объекта
Цель: продать квартиру.

```
Создан → Подготовка → Лиды → Показы → Сделка → Продано → Постсервис → [Отмена]
```

1. **Создан** — объект создан на этапе интервью, но не активен
2. **Подготовка** — фото, описание, публикация объявлений
3. **Лиды** — приём заявок от покупателей
4. **Показы** — просмотры и переговоры
5. **Сделка** — подписание ДКП
6. **Продано** — сделка закрыта
7. **Постсервис** — отзыв, доп. услуги
8. **Отмена** — снято с продажи

## Классификация объектов (автоматическая)

Система автоматически определяет класс квартиры по параметрам:

| Класс | Условия |
|-------|---------|
| **Старфонд** | Год постройки ≤ 1980 (автоматически, без исключений) |
| **Эконом** | 1981-1997, панель, потолки < 2.6м, слабая локация (2+ условия) |
| **Комфорт** | ≥ 1998, потолки 2.6-2.99м, ≤ 20 этажей, есть лифт, нормальная локация |
| **Комфорт+** | ≥ 2010, кирпич/монолит, потолки 3.0-3.19м, ≤ 12 этажей, паркинг, закрытая территория |
| **Бизнес** | ≥ 2015, кирпич/монолит, потолки ≥ 3.2м, ≤ 9 этажей, ≤ 4 кв. на площадке, подземный паркинг, панорамное остекление, Face ID, премиум лобби (ВСЕ 13 условий) |

## Стратегии продажи (12 штук, выбирается автоматически)

| Код | Название | Когда применяется |
|-----|----------|-------------------|
| S1 | Рыночная продажа | Цена ±5% от рынка, нет срочности, есть доверие |
| S2 | Срочная продажа | Срочность до 30 дней, готовность к скидке 5-10% |
| S3 | Продажа → Покупка | Продавец планирует покупку после продажи |
| S4 | Завышенные ожидания | Цена выше рынка >10%, продавец защищает цену |
| S5 | Ограниченное вовлечение | Нет эксклюзива, продавец сомневается |
| S6 | Инвестиционный выход | Продавец-инвестор, решения по цифрам |
| S7 | Юридически сложный | Залог, обременения, доли, наследство |
| S8 | Низкая ликвидность | Плохая локация, старфонд, узкий рынок |
| S9 | Альтернативный выход | Крайняя срочность, несколько ЛПР |
| S10 | Отказ от объекта | Нет доверия, высокие риски, нарушение правил |
| S11 | Дорогой объект | Цена в топ-20% по классу и району |
| S12 | Тест рынка | Спорная цена, мало аналогов, нужна проверка |

Правило: у объекта всегда ОДНА активная стратегия.

## Дополнительный функционал

- **Ипотечный калькулятор** — расчёт платежей, сравнение программ банков
- **Курсы обучения** — материалы для брокеров (видео, чек-листы), прогресс прохождения
- **Уведомления** — системные, от куратора, по сделкам, бонусам
- **Задачи** — личный список задач брокера
- **Баланс и платежи** — история операций, комиссии
- **Лид-формы** — создание форм для сбора заявок, распределение между брокерами
- **Кастомные воронки** — брокер может создавать свои этапы воронки
- **Кастомные поля** — дополнительные поля для продавцов/объектов
- **Календарь событий** — показы, встречи, звонки
- **Экспорт данных** — выгрузка в Excel/CSV
- **AI-генерация** — описание объекта и стратегия через DeepSeek API
- **Архив** — мягкое удаление с возможностью восстановления
