## 4. РОЛИ ПОЛЬЗОВАТЕЛЕЙ И ИХ ФУНКЦИИ

Система Casa Pro поддерживает 5 ролей. Приоритет разработки: ADMIN + BROKER (текущий этап), остальные роли — следующий этап.

**Источник приоритета:** Заказчик, Группа, 21.01.2026, 17:21:

### 4.1. ADMIN — Администратор

Полный контроль над системой. Роль Заказчика (Алике).

**Доступные разделы:**
- Главная (Dashboard) — общая статистика по всей системе
- Сделки (CRM) — видит все сделки всех брокеров
- Стратегии (CASA) — справочник 12 стратегий продажи
- Мои объекты — все объекты в системе
- Новостройки (Каталог ЖК, Шахматка)
- Ипотека — калькулятор и управление программами
- Профиль
- Формы — управление лид-формами
- Настройки — воронки, кастомные поля
- Архив
- Управление (только ADMIN): Пользователи, Курсы, Все проекты, Настройки AI

**Уникальные функции ADMIN:**
- Создание и управление пользователями любых ролей
- Доступ к данным всех брокеров (клиенты, сделки, объекты)
- Управление ипотечными программами (добавление/редактирование/удаление)
- Настройки системы
- Назначение кураторов брокерам

### 4.2. BROKER — Брокер (основная рабочая роль)

Главный пользователь системы. Ведёт клиентов от первого контакта до закрытия сделки.

**Доступные разделы:**
- Главная — личная статистика (сделки, клиенты, баланс, график продаж)
- Сделки (CRM) — Kanban-доска + табличные списки по продавцам и объектам
- Стратегии (CASA) — справочник стратегий (только просмотр)
- Мои объекты — список своих объектов недвижимости
- Новостройки (Каталог ЖК, Шахматка)
- Ипотека — калькулятор (без редактирования программ)
- Профиль (баланс, обучение, аналитика, уведомления, настройки)
- Формы — свои лид-формы
- Настройки — свои воронки и кастомные поля
- Архив

**Функции брокера:**
- Создание продавцов (карточки клиентов-продавцов) — форма `Копия Crm Lead Incoming Ui.md`
- Проведение интервью, сбор данных о продавце и объекте
- Создание объектов недвижимости — форма `Casa Pro Object Card.md`
- Ведение продавца по воронке: Контакт → Интервью → Стратегия → Договор → [Отказ]
- Ведение объекта по воронке: Создан → Подготовка → Лиды → Показы → Сделка → Продано → Постсервис → [Отказ]
- Создание покупателей и организация показов
- Оформление сделок, фиксация комиссии
- Загрузка документов и фото объектов
- Архивирование контактов (не удаление)
- Связь с клиентами через WhatsApp из CRM
- Прохождение курсов обучения
- Видит только свои данные

### 4.3. DEVELOPER — Застройщик (следующий этап)

Управляет жилыми комплексами (новостройками) и квартирами в них.

**Доступные разделы:**
- Главная
- Сделки (CRM)
- Новостройки (Каталог ЖК, Шахматка)
- Ипотека
- Профиль
- Настройки
- Архив

**Уникальные функции:**
- Создание и редактирование проектов (ЖК): название, город, район, класс, статус строительства, фото, видео
- Управление квартирами в своих ЖК (шахматка): номер, этаж, комнаты, площадь, цена, статус
- Видит только свои проекты

**Источник:** Заказчик, Группа, 21.01.2026, 17:21: «потом другие кабинеты» — застройщик отложен на следующий этап.

---

### 4.4. REALTOR — Риелтор (следующий этап)

Работает аналогично брокеру, но может быть частью агентства.

**Доступные разделы:**
- Главная, Сделки (CRM), Мои объекты, Клиенты, Новостройки, Ипотека, Профиль, Настройки, Архив

**Функции:**
- Те же что у BROKER: создание продавцов, объектов, покупателей, сделок
- Видит только свои данные
- Может быть подчинён агентству (AGENCY видит его статистику)

---

### 4.5. AGENCY — Агентство недвижимости (следующий этап)

Управляет командой риелторов.

**Доступные разделы:**
- Главная, Сделки (CRM), Команда, Мои объекты, Клиенты, Новостройки, Ипотека, Профиль, Формы, Настройки, Архив

**Уникальные функции:**
- Раздел «Команда»: просмотр риелторов, их сделок, продавцов, объектов
- Создание риелторов (REALTOR) в своей команде
- Видит данные своих риелторов
